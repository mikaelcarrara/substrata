Getting started

Substrata is designed to be consumed, not imposed.
You can adopt it incrementally, partially, or as a full design foundation.

Manual Installation

Download the latest CSS bundles directly:

git clone https://github.com/mikaelcarrara/substrata.git

Include the src/substrata.css file in your HTML or CSS imports.

Import everything

@import "path/to/substrata/src/substrata.css";

This includes tokens, base styles, and reference components.

Or import only what you need

@import "path/to/substrata/src/tokens/colors.css";
@import "path/to/substrata/src/tokens/spacing.css";
@import "path/to/substrata/src/tokens/typography.css";
@import "path/to/substrata/src/tokens/radius-and-borders.css";
@import "path/to/substrata/src/base.css";

Using with Tailwind CSS

Substrata does not replace Tailwind.
It provides design tokens that Tailwind can consume.

Import tokens

@import "path/to/substrata/src/substrata.css";
@tailwind base;
@tailwind components;
@tailwind utilities;

Map tokens in 'tailwind.config.js'

export default {
  theme: {
    extend: {
      colors: {
        brand: {
          300: "var(--brand-300)",
          500: "var(--brand-500)",
          700: "var(--brand-700)",
        },
        neutral: {
          0: "var(--neutral-0)",
          900: "var(--neutral-900)",
        },
      },
      spacing: {
        1: "var(--space-1)",
        2: "var(--space-2)",
        3: "var(--space-3)",
        4: "var(--space-4)",
      },
      borderRadius: {
        md: "var(--radius-md)",
      },
    },
  },
};

Tailwind utilities reflect the same design decisions defined in Substrata.

Using with CSS-in-JS

Substrata acts purely as a token layer.

Import tokens globally

import "path/to/substrata/src/tokens.css";

Use tokens inside components

const Button = styled.button`
  padding: var(--space-3) var(--space-5);
  background: var(--brand-500);
  color: var(--neutral-0);
  border-radius: var(--radius-md);
`;

No coupling.
No runtime dependency.
No framework assumptions.
