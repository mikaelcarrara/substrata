What is Substrata?

Substrata is a design token system built on CSS Variables, providing a stable, framework-agnostic foundation beneath the UI. It defines design decisions as constraints - not implementations.

Unlike UI libraries that couple design to components, Substrata defines only primitives: colors, spacing, typography, and other core decisions expressed as semantic, versionable tokens.

This separation preserves design intent across platforms (Web, iOS, Android) and implementation layers (React, Vue, plain HTML, Sass, Tailwind, CSS-in-JS), without rewriting the design system as stacks evolve.

Why Substrata exists

Most design systems are tightly coupled to specific frameworks or component models. When the stack changes, the system must be rebuilt, causing design decisions to be duplicated, lost, or drift across products.

Substrata addresses this by treating design decisions as infrastructure - stable, explicit, framework-agnostic contracts that outlive any single implementation.

  • Tokens as contracts

    Design decisions become stable, versionable interfaces between design and code.

  • Framework independence

    Consume tokens in Tailwind, Sass, CSS-in-JS, or plain CSS without coupling to a specific stack.

  • Automation-ready

    Explicit tokens enable safe automation, including AI-assisted generation, refactoring, and validation.

  • Shared design language

    A common vocabulary that designers, developers, and tools can interpret and apply consistently.

Substrata is composed of independent layers.

Substrata is structured into three decoupled layers. Each layer can evolve independently, without impacting the others.

  • Tokens define design decisions as CSS Variables: colors, spacing, typography, and core primitives. They are semantic, stable, versionable, and platform-agnostic.

  • Base styles apply tokens to native HTML elements, establishing predictable defaults and consistent rendering, without adding abstractions or components.

  • Components (optional) show reference patterns for consuming tokens in real interfaces. They demonstrate usage, not prescribe a UI library.

Philosophy

Design decisions are infrastructure.

  • Design decisions should be explicit, not implicit
  • Consistency should survive framework migrations
  • Tokens are stable contracts, not arbitrary values
  • Substrata is infrastructure - the foundation beneath the UI

This philosophy aligns with the W3C Design Tokens Community Group model and is designed for long-term evolution across platforms and teams.
