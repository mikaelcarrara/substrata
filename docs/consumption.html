Consumption

Substrata defines what. Consumers decide how.

Substrata tokens can be consumed by any styling approach. The integration model is intentionally framework-agnostic. The examples below apply the same design decisions using different consumption strategies.

Plain CSS

.card {
  padding: var(--space-4);
  background: var(--color-surface);
  color: var(--color-text-primary);
  border-radius: var(--radius-md);
}

Utility-first (Tailwind)

tailwind.config.js
module.exports = {
  theme: {
    spacing: { 4: 'var(--space-4)' },
    colors: {
      surface: 'var(--color-surface)',
      text: 'var(--color-text-primary)',
    },
    borderRadius: { md: 'var(--radius-md)' },
  },
};

CSS Preprocessors (Sass / Less)

Preprocessors do not transform tokens. They only organize consumption. 

.card {
  padding: var(--space-4);
  background: var(--color-surface);
  border-radius: var(--radius-md);
}

CSS-in-JS

// styled-components / emotion
const Card = styled.div`
  padding: var(--space-4);
  background: var(--color-surface);
  border-radius: var(--radius-md);
`;
// vanilla-extract
export const card = style({
  padding: 'var(--space-4)',
  background: 'var(--color-surface)',
  borderRadius: 'var(--radius-md)',
});

AI / Automation

Generate a Card component using only:
- space-4
- color-surface
- radius-md

Do not introduce hardcoded values.
Output plain CSS.
